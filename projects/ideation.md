로컬 환경에서 **비용 0원**으로 구현 가능하면서도, 카카오엔터테인먼트가 요구하는 **추천/랭킹 모델 역량**을 가장 잘 보여줄 수 있는 커머스 기반 MVP 프로젝트를 제안합니다.

이 프로젝트는 **H&M의 실제 구매 데이터**를 활용하여, 특정 유저에게 "오늘 어떤 상품을, 몇 시에 추천(푸시)할 것인가"를 결정하는 로직을 로컬에서 완결하는 것을 목표로 합니다.

## 🛍️ 프로젝트명: Local-Helix (Commerce Edition)

**"유저 구매 이력 기반의 시계열 푸시 타겟팅 및 랭킹 엔진"**

### 1. 데이터셋 및 환경 (Local Setup)

* **데이터:** [Kaggle H&M Personalized Fashion Recommendations](https://www.google.com/search?q=https://www.kaggle.com/competitions/h-and-m-personalized-fashion-recommendations/data)
* **이유:** 3,100만 건의 트랜잭션 데이터가 있어 대용량 처리 경험을 쌓기 좋고, 구매 시각(Timestamp)이 있어 '푸시 시점 최적화' 실험이 가능합니다.


* **환경:** 여러분의 노트북 (Python + DuckDB)
* **DuckDB:** 별도의 서버 설치 없이 `pip install duckdb`만으로 수천만 건의 데이터를 SQL로 초고속 처리할 수 있습니다.



---

### 2. MVP 핵심 워크플로우 (End-to-End)

#### **1단계: DuckDB를 활용한 SQL 피처 엔지니어링**

실제 현업에서는 데이터 추출이 업무의 70%입니다. 로컬 파일(CSV/Parquet)을 DuckDB로 연결해 SQL 능력을 증명하세요.

* **유저별:** 최근 4주간 평균 구매 시간대, 선호 카테고리, 구매 주기(Recency).
* **상품별:** 최근 1주일간 판매 순위(Popluarity), 상품별 주된 구매 시간대.
* **결과물:** `user_features`, `item_features` 테이블 생성.

#### **2단계: 후보군 생성 (Candidate Generation)**

모든 유저에게 모든 상품(10만 개 이상)을 연산할 수 없습니다. 1차 필터링 로직을 구현합니다.

* **로직:** 유저가 최근 구매한 것과 유사한 상품(Item-based CF) + 현재 가장 인기 있는 상품 Top 50개 추출.
* **핵심:** 이 과정을 통해 '효율적인 추천 시스템 설계' 역량을 보여줍니다.

#### **3단계: LightGBM 랭킹 모델 (The Ranking)**

추출된 50개 후보 중 유저가 실제로 구매할 확률이 가장 높은 순서대로 줄을 세웁니다.

* **학습:** (유저, 상품, 시간) 조합을 입력했을 때 구매 여부(1 or 0)를 예측하도록 학습.
* **평가:** 유저가 실제로 구매한 시각과 모델이 가장 높은 점수를 준 시각을 비교하여 **Hit Rate** 산출.

#### **4단계: 로컬 서빙 시뮬레이션 (The Push)**

* **기능:** `predict(user_id)` 함수를 실행하면, 해당 유저의 과거 패턴을 분석해 **"오늘 오후 7시에 '스키니 진' 푸시를 보내는 것이 구매 확률이 가장 높음"**이라는 결과를 텍스트 파일이나 SQLite로 저장합니다.

### 3. 카카오엔터 지원용 '필살기' 포인트

단순히 "추천 모델 만들었다"고 하지 마시고, 이 프로젝트를 통해 아래의 내용을 포트폴리오에 쓰세요.

1. **"DuckDB를 활용해 3,000만 건의 데이터를 로컬 환경에서 SQL로 전처리하며 인프라 비용을 최소화함"** (비용 및 SQL 역량)
2. **"H&M의 구매 시점 데이터를 활용해, 카카오페이지의 '헬릭스 푸시'와 유사한 시각 기반의 추천 최적화 로직을 실험함"** (직무 유사성)
3. **"단순 추천이 아니라, 모델의 예측값(Prob)을 기준으로 푸시 우선순위를 결정하는 랭킹 모델 시스템을 설계함"** (추천/랭킹 전문성)
